Getting Started

In order to test our UMB Messenger, you will need to run it alongside the backend.
Note: If the school wishes to use their own backend, it's only a matter of changing a few components in the client.



To start, ensure that you have Intellij IDEA installed (community edition is fine). Because both our services are written in JAVA, you do not need to download anything else. More information on Required Tools.

Backend Setup

Installation -

Download the backend host from our github master repo.

Download Github Repo Here

Running the Host-

When you build our maven project with Intellij, press the run button. To ensure that the service is working as expected, visit the following: http://localhost:8000/health

If the message "Up" is displayed on the browser, this means that the backend is running properly.

Troubleshoot -

If you're coming across any errors do the following:

  • • Ensure your intellij is set to JDK 17. You can also download it directly from Intellij.
  • • Make sure you are running on port 8000. To set that up, check if the application.properties contains: "server.port=8000". If it does not, add it.
  • • Check the POM file. Sometimes it is just a dependency issue.
  • • Try rebuilding the maven project.
  • • In Intellij, sometimes just invalidating the cache fixes the problem. To invalidate the cache do the following:

    File -> Invalidate Cache -> Invalidate and restart

Client Setup

Installation -

Download the client from our github master repo.

Download Github Repo Here

Running the Client-

When you build our maven project with Intellij, press the run button. You'll know it's running once a program with the label UMB Messenger is running.

Ensure that the host is running along side the client in order for it to work. After you verified that that the backend is "Up", continue the signup process.

Troubleshoot -

If you're coming across any errors do the following:

  • • Ensure your intellij is set to JDK 17. You can also download it directly from Intellij.
  • • Check the POM file. Sometimes it is just a dependency issue.
  • • Try rebuilding the maven project.
  • • If you get any atsign errors, make sure to import the JAR file.

    • Steps: File -> Project Structure -> Project Settings -> Modules -> Dependencies -> (+) and add the jar files from the lib folder.

  • • In Intellij, sometimes just invalidating the cache fixes the problem. To invalidate the cache do the following:

    File -> Invalidate Cache -> Invalidate and restart
